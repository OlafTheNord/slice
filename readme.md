Твоя задача: создать небольшую систему, которая умеет:

Хранить список счетов.
Выполнять транзакции (пополнение, снятие).
Применять разные типы комиссий к транзакциям.
Детальные требования:

Структура Account:

Создай структуру Account с полями:
ID (тип int)
Owner (тип string)
Balance (тип int) - баланс будем хранить в копейках/центах, чтобы избежать проблем с float.
Интерфейс Transaction:

Создай интерфейс Transaction с одним методом Apply(account *Account).
Этот метод должен принимать указатель на Account и изменять его баланс.
Реализации Transaction:

Создай две структуры, реализующие интерфейс Transaction:
Deposit (пополнение) с полем Amount int. Его метод Apply должен увеличивать баланс счета на Amount.
Withdrawal (снятие) с полями Amount int и Fee int (комиссия). Его метод Apply должен уменьшать баланс счета на Amount + Fee.
Хранилище Bank:

Создай структуру Bank, которая будет управлять всеми счетами.
У нее должно быть одно поле: Accounts map[int]*Account. Это будет map, где ключ — ID счета, а значение — указатель на сам счет. (Использование указателя здесь критично, чтобы изменения, сделанные через транзакции, отражались в хранилище).
Добавь в Bank метод Execute(accountID int, tx Transaction) error.
Этот метод должен найти счет по accountID в b.Accounts.
Если счет не найден, вернуть ошибку (используй fmt.Errorf("account with id %d not found", accountID)).
Если счет найден, вызвать метод tx.Apply() для этого счета.
Если все прошло успешно, вернуть nil.
Функция main:

Создай новый Bank.
Создай несколько счетов (Account) и добавь их в bank.Accounts.
Создай несколько транзакций (Deposit, Withdrawal).
Используя метод bank.Execute(), примени эти транзакции к счетам.
В конце выведи на печать итоговое состояние всех счетов, чтобы убедиться, что балансы изменились правильно.
Протестируй случай с несуществующим счетом и убедись, что твоя функция Execute корректно возвращает ошибку.
